Molecular Dynamics

Overview

Molecular dynamics (MD) propagates the equations of motion for a molecular system, generating a time-ordered trajectory from which thermodynamic properties, conformational flexibility, and dynamical processes can be extracted. MAPLE integrates ML potentials with a symplectic Velocity Verlet integrator and provides three statistical ensembles covering the most common simulation conditions.

MD simulations in MAPLE are invoked with the #md directive. The ensemble is selected via the ensemble parameter. All three ensembles share a common integrator and output infrastructure.

Available Ensembles

Ensemble Keyword Fixed Quantities Description
NVE nve N, V, E Microcanonical ensemble. Energy-conserving reference simulation; no thermostat or barostat. Default.
NVT nvt N, V, T Canonical ensemble. Constant temperature via Langevin or V-rescale thermostat.
NPT npt N, P, T Isothermal-isobaric ensemble. Constant temperature and pressure; requires periodic boundary conditions.

Quick Usage

To run an MD simulation, add the #md directive to your input file. The ensemble is specified with the ensemble parameter:

#md(ensemble=nvt, temperature=300, steps=100000)

If no ensemble is specified, NVE is used by default:

#md

Input Example

A complete NVT equilibration of a water molecule at 300 K for 100 ps:

#model=ani2x
#md(ensemble=nvt, temperature=300, steps=100000, timestep=1.0, thermostat=langevin)
#device=cpu

O    0.000000    0.000000    0.117300
H    0.000000    0.757200   -0.469200
H    0.000000   -0.757200   -0.469200

Common Parameters

These parameters are accepted by all three ensembles. Ensemble-specific parameters are documented on the individual ensemble pages.

Parameter Type Default Description
ensemble str nve Statistical ensemble: nve, nvt, or npt.
timestep float ensemble-dependent Integration time step in femtoseconds. Default: 0.25 fs (NVE), 1.0 fs (NVT), 0.5 fs (NPT).
steps int ensemble-dependent Total number of integration steps. Default: 400000 (NVE), 100000 (NVT), 10000 (NPT).
temperature float 300.0 Target temperature in Kelvin. Used for velocity initialization in all ensembles and for thermostat control in NVT/NPT.
traj_every int 100 Write a trajectory frame every N steps.
log_every int 100 Write a thermodynamic log entry every N steps.
init_velocities bool True Initialize velocities from a Maxwell–Boltzmann distribution at the target temperature.
init_from str None Path to a *_md_final.xyz file from a previous run. Loads positions and velocities to continue a simulation.
resume bool False Resume an interrupted simulation, appending to existing output files and offsetting the step counter.
remove_com bool True Periodically remove center-of-mass translational momentum to eliminate floating-point drift.
random_seed int None Random seed for reproducible velocity initialization and thermostat noise. If None, a random seed is used.
verbose int 1 Output verbosity (0 = minimal, 1 = standard thermodynamic output, 2 = debug).

Output Files

Every MD run produces the following output files, where * is the input file base name:

File Contents
*_md_traj.xyz Full trajectory in extended XYZ format. Each frame includes atomic symbols, positions (Å), and velocities (Å/fs), plus a comment line with step number, time, energy, and temperature.
*_md.dat Thermodynamic log. Tab-separated columns: step, time (ps), total energy (Eh), potential energy (Eh), kinetic energy (Eh), temperature (K), and — for NPT — pressure (bar) and volume (Å³).
*_md_final.xyz Last frame of the trajectory, including velocities. Suitable as input to init_from for continuing or chaining simulations.

Recommended Workflow

A typical ML-MD protocol for a molecular system proceeds in three stages:

  1. Geometry optimization — Relax the input structure with #opt before starting MD to remove large forces that can destabilize the integration.
  2. NVT equilibration — Run NVT for at least 50–100 ps to bring the system to the target temperature. Save the final frame with init_from.
  3. NVE or NVT production — Use the equilibrated structure as input. NVE is preferred for computing dynamical properties; NVT is appropriate for thermodynamic sampling.
# Step 1: optimization
#model=ani2x
#opt

O    0.000000    0.000000    0.117300
H    0.000000    0.757200   -0.469200
H    0.000000   -0.757200   -0.469200
# Step 2: NVT equilibration (water_opt_opt.xyz produced by step 1)
#model=ani2x
#md(ensemble=nvt, temperature=300, steps=100000)

XYZ water_opt_opt.xyz
# Step 3: NVE production from equilibrated structure
#model=ani2x
#md(ensemble=nve, steps=400000, init_from=water_md_nvt_md_final.xyz)

XYZ water_opt_opt.xyz
Important

NPT simulations require a periodic simulation cell. Set the cell dimensions using the #pbc directive before running NPT. Isolated molecules without PBC must use NVE or NVT.
