NPT Molecular Dynamics

Overview

The NPT ensemble (isothermal-isobaric ensemble) holds particle number (N), pressure (P), and temperature (T) constant. Both a thermostat and a barostat are active simultaneously: the thermostat regulates kinetic energy (temperature), while the barostat periodically rescales the simulation cell to maintain the target pressure. NPT is the appropriate ensemble for condensed-phase simulations where the system must relax to its equilibrium density, and for computing thermodynamic properties such as density, compressibility, and enthalpy.

NPT requires periodic boundary conditions. Isolated gas-phase molecules must use NVE or NVT instead.

MAPLE provides two thermostat choices and two barostat choices for NPT, which can be combined independently:

Thermostats

  • V-rescale (default for NPT): stochastic velocity rescaling (Bussi et al., 2007). Uniform velocity scaling with a stochastic correction. Produces the correct NVT marginal distribution.
  • Langevin: per-atom stochastic friction using the BAOAB scheme. Correct canonical distribution; ergodic. Higher friction can interfere with slow collective motions in condensed phases.

Barostats

  • C-rescale (default for NPT): stochastic cell rescaling (Bernetti & Bussi, 2020). Deterministic Berendsen rescaling augmented with a stochastic noise term to produce the correct NPT distribution. Recommended for production simulations.
  • Berendsen: deterministic cell rescaling. Does not produce the correct NPT ensemble and suppresses pressure fluctuations. Use only for rapid pre-equilibration of cell dimensions before switching to C-rescale.

Parameters

Parameter Type Default Description
timestep float 0.5 Integration time step in femtoseconds. Smaller than NVT to ensure stability with cell rescaling operations.
steps int 10000 Total number of integration steps (= 5 ps at 0.5 fs default). Extend to at least 200000 steps for production density calculations.
temperature float 300.0 Target temperature in Kelvin.
pressure float 1.0 Target pressure in bar.
thermostat str v-rescale Thermostat algorithm. Options: v-rescale, langevin.
barostat str c-rescale Barostat algorithm. Options: c-rescale (correct NPT ensemble), berendsen (pre-equilibration only).
friction float 0.001 Langevin friction coefficient in 1/fs. Used only when thermostat=langevin.
tau_t float 200.0 Temperature coupling time constant τT in femtoseconds for V-rescale. Slightly larger than the NVT default to avoid over-coupling in the NPT ensemble.
tau_p float 2000.0 Pressure coupling time constant τP in femtoseconds. Large value (2 ps) accounts for the noise in ML potential stress tensors; do not set below 500 fs.
compressibility float 4.5e-5 Isothermal compressibility in 1/bar. Default is the compressibility of liquid water at 300 K and 1 bar. Adjust for other solvents or materials.
traj_every int 100 Write a trajectory frame every N steps.
log_every int 100 Write a thermodynamic log entry every N steps. The log includes pressure and volume in addition to the standard energy and temperature columns.
init_velocities bool True Initialize velocities from a Maxwell–Boltzmann distribution at temperature.
init_from str None Path to a *_md_final.xyz file from a previous run.
resume bool False Resume an interrupted simulation.
random_seed int None Integer seed for reproducible velocity initialization and stochastic thermostat/barostat noise.
verbose int 1 Output verbosity (0 = minimal, 1 = standard).

Input Example

NPT density equilibration of a periodic water box at 300 K and 1 bar using the recommended V-rescale + C-rescale combination:

#model=maceoff23m
#md(ensemble=npt, temperature=300, pressure=1.0, steps=200000, timestep=0.5)
#pbc(18.6, 18.6, 18.6)
#device=gpu0

XYZ water_box.xyz

Pre-equilibration with Berendsen barostat for rapid cell relaxation, followed by production with C-rescale:

# Pre-equilibration (berendsen_equil.inp)
#model=maceoff23m
#md(ensemble=npt, temperature=300, pressure=1.0, steps=20000,
    barostat=berendsen, tau_p=500.0)
#pbc(18.6, 18.6, 18.6)
#device=gpu0

XYZ water_box.xyz
# Production (production.inp)
#model=maceoff23m
#md(ensemble=npt, temperature=300, pressure=1.0, steps=400000,
    barostat=c-rescale, init_from=water_box_berendsen_equil_md_final.xyz)
#pbc(18.6, 18.6, 18.6)
#device=gpu0

XYZ water_box.xyz

Choosing a Barostat

Barostat Correct Ensemble Pressure Fluctuations When to Use
c-rescale Yes Physically correct All production NPT simulations. Default recommendation.
berendsen No Suppressed Rapid initial cell equilibration only. Do not use for computing thermodynamic averages.
Important

NPT requires a periodic simulation cell defined with the #pbc directive. Attempting to run NPT on a non-periodic system will raise an error. See the Coordinates page for the #pbc syntax.

Note

The default tau_p=2000.0 fs is intentionally long. ML potentials typically have higher stress-tensor noise than classical force fields, so aggressive pressure coupling (short τP) can lead to cell instability. If the density converges slowly, do not decrease τP below 500 fs.
