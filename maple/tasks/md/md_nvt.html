NVT Molecular Dynamics

Overview

The NVT ensemble (canonical ensemble) holds particle number (N), volume (V), and temperature (T) constant. A thermostat couples the system to a heat bath, maintaining the target temperature throughout the simulation. NVT is the standard choice for temperature equilibration and for thermodynamic sampling at a fixed temperature.

MAPLE provides two NVT thermostats:

  • Langevin (default): per-atom stochastic friction and random forces using the BAOAB splitting scheme (Leimkuhler & Matthews, 2013). Produces the correct canonical ensemble; ergodic by construction. The standard choice for most NVT simulations.
  • V-rescale: stochastic velocity rescaling (Bussi et al., 2007). Scales all velocities uniformly by a single factor with a stochastic correction term, producing the correct canonical ensemble without per-atom noise. Less perturbative to dynamical properties than Langevin, but slightly less robust far from equilibrium.

Parameters

Parameter Type Default Description
timestep float 1.0 Integration time step in femtoseconds. 1.0 fs is safe for most organic molecules with ML potentials when C–H bond stretching is the fastest motion.
steps int 100000 Total number of integration steps (= 100 ps at 1 fs default).
temperature float 300.0 Target temperature in Kelvin.
thermostat str langevin Thermostat algorithm. Options: langevin, v-rescale.
friction float 0.001 Langevin friction coefficient γ in 1/fs. Controls the coupling strength to the heat bath. Used only when thermostat=langevin.
tau_t float 100.0 Temperature coupling time constant τT in femtoseconds for V-rescale. Larger values give weaker coupling; smaller values follow the target temperature more closely. Used only when thermostat=v-rescale.
traj_every int 100 Write a trajectory frame every N steps.
log_every int 100 Write a thermodynamic log entry every N steps.
init_velocities bool True Initialize velocities from a Maxwell–Boltzmann distribution at temperature.
init_from str None Path to a *_md_final.xyz file. Loads positions and velocities from a previous run.
resume bool False Resume an interrupted simulation, appending to existing output files.
random_seed int None Integer seed for reproducible velocity initialization and thermostat noise.
verbose int 1 Output verbosity (0 = minimal, 1 = standard).

Input Example

NVT equilibration at 300 K using the default Langevin thermostat:

#model=ani2x
#md(ensemble=nvt, temperature=300, steps=100000, thermostat=langevin)
#device=cpu

O    0.000000    0.000000    0.117300
H    0.000000    0.757200   -0.469200
H    0.000000   -0.757200   -0.469200

NVT run with V-rescale thermostat and tighter temperature coupling:

#model=ani2x
#md(ensemble=nvt, temperature=300, steps=200000, thermostat=v-rescale, tau_t=50.0)
#device=cpu

XYZ water_opt_opt.xyz

Choosing a Thermostat

Thermostat Correct Ensemble Dynamics When to Use
langevin Yes Perturbed (per-atom noise) General-purpose equilibration; robust far from equilibrium; default recommendation.
v-rescale Yes Less perturbed (global scaling) When dynamical properties (diffusion, spectra) must be preserved and the system is already near equilibrium.
Note

The Langevin friction coefficient friction=0.001 1/fs is intentionally weak. This mild coupling ensures correct canonical sampling while minimally perturbing inertial dynamics. Only increase it (e.g., to 0.01) if temperature equilibration is very slow.
